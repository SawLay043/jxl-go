# Product Guidelines: jxl-go

## Prose Style
- **Formal/Narrative:** The overall prose style for project documentation should be descriptive and narrative-driven, aiming to provide a high level of context and accessibility to a wide audience of developers and enthusiasts.

## Code Documentation Style
- **Explanatory/In-depth:** We prioritize detailed, multi-line comments that go beyond simple function descriptions. Documentation should explain complex logic, algorithms, and technical trade-offs to aid long-term maintainability and understanding.

## UX Principles
- **API Simplicity First:** We prioritize a clean and intuitive API that makes it easy for Go developers to integrate JXL decoding into their projects with minimal learning curve.
- **Performance-Centric Design:** Speed and resource efficiency are core considerations in every design decision, ensuring the library remains competitive and useful in high-throughput environments.
- **Correctness and Reliability:** We ensure the library is robust, strictly adheres to the JPEG XL specification, and handles errors gracefully to provide a predictable and trustworthy user experience.

## Error Handling
- **Descriptive Errors:** Our approach to error handling is to return descriptive errors that provide clear explanations of what went wrong, aiding in debugging and integration.
